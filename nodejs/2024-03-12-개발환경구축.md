## node.js 개발환경 구축하기   

### npm

+ **npm이란** 노드 패키지 매니져의 약자로 노드 구동시 사용되는 패키지들을 관리하는 모듈이다.   
   
  웹 프로그램들은 구동할 때 필요로 하는 패키지들이 다 다르기에 노드에서 따로 정하지 않고 모듈을 만들어서

  해당 웹 프로그램에 적용시킨다.   

  > npm을 설치하려면 원하는 디렉터리에서 cmd로 **node install npm**을 쳐주어야 한다

### package.json

+ **package.json이란** 해당 노드 파일에서 사용되는 패키지와 모듈들을 관리하는 파일을 가리킨다. 

  그냥 아무렇게 너저분하게 파일이 널려있어 그것을 노드 패키지 매니저가 일일이 찾아서 도킹시켜주는 것보다

  훨씬 빠르고 버전확인이나 수정이 용이하다

  > 처음 package 파일을 만들 때는 **npm init**을 cmd로 쳐주어야한다.

  > 옵션으로 -y을 하면 노드제이슨에서 기본으로 제공하는 패키지를 만들어준다

### dependencie & devdependencie

+ 모듈과 패키지의 **의존성**을 표시하는 패키지의 속성이다.

  > 의존성이 필요한 이유는 프로그램은 계속 업데이트 되고 수정되면서 모듈도 같이 수정이 일어난다

  > 그러면 프로그램이 참조해야할 모듈과 패키지를 알아야 하고 더 나아가 하위호환을 지원하기 위해

  > 패키지 파일이 사용된다.

### nodemon

+ **nodemon이란** 노드를 구동중일 때, 구동 중인 파일이 수정이 일어나면 바로 그것을 반영하여 다시 시작하게 해주는 모듈을 말한다

  > 사용하기 위해선 cmd에서 **npm install nodemon**이라 쳐주어야 한다.

### babel

+ **babel이란** 최신 ES6, ES5 코드를 노드가 읽을 수 있는 commonJS으로 변환시키는 프로그램이다.   
  
  > 해당 프로그램이 필요한 이유는 어떤 브라우저는 commonJS를 기준으로 돌아가 최신 버전으로 작성한 코드가

  > 읽히지 않는 문제가 발생하기 때문이다.

## module

+ **모듈이란** 자주 사용되고 버전관리와 수정을 쉽게하기 위해 같은 수단을 쓰는 함수나 객체들을 모아둔 단위를 말한다.

  노드에서는 이 모듈을 자바스크립트 파일에 임포트 시켜서 사용한다.

  이 때 ES6이냐 commonJS냐에 따라 임포트 방법이 달라진다.

  ### commonJS

  + 모듈 내보내기 : **export**

  + 모듈 받아쓰기 : **require("모듈명, 모듈존재파일")**

  ### ES6

  + 모듈 내보내기 : **defalut export**

  + 모듈 받아쓰기 : **import {} from "모듈명, 모듈존재파일"**
